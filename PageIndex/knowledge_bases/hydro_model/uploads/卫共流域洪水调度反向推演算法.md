# 卫共流域洪水调度反向推演算法  

# 一、前端调度目标设置  

调度目标:元村集  洪峰流量 $<2500\mathrm{m}^{3}/\mathsf{s}$   
优化方向:在满足调度目标前提下,尽量减少蓄滞洪区启用和淹没。附加约束:  

1、水库调洪水位约束  

$\odot$ 最大库水位 $<=$ 防洪高水位$\bigcirc$ 最大库水位 $<=$ 设计洪水位○最大库水位 $<=$ 校核洪水位  

# 2、其他约束  

闸站过流能力约束河道过流能力约束蓄滞洪区滞洪水位约束  

# 二、后端算法:  

# 1、约束条件  

(1)调度目标约束(非硬性约束,调度目标无法达到时,给出 最接近调度目标的结果);  

(2)附加约束条件(硬性约束,但需遵守客观规律: $\textcircled{1}$ 为确保水库安全,库水位约束条件必须遵守,通过减少洪水调蓄一般均能达到,但也存在由于洪水来势急,溢洪道泄洪洞全开由于过流能力限制,导致最高水位超过库水位约束的情况);  

(3)闸站过流能力约束、河道过流能力约束、蓄滞洪区设计滞洪水位约束(硬性约束);  

# 2、模型算法  

# (1)边界条件和初始条件  

和洪水调度(正向推演)一样,洪水来源可直接指定洪水,也可采用降雨计算洪水;  

# (2)水库调蓄和泄洪计算  

1)调蓄算法  

目标为在调洪库容下尽量减少控泄流量和下泄洪量。  

当入库总洪量小于调洪库容时,闸门全程关闭,下泄为 0;  

当入库总洪量大于调洪库容时,采用消峰调蓄算法;将调洪库容用到刀尖上,并用满调洪库容:根据入库洪水流量过程和可调蓄的库容(调蓄约束水位-初始水位),进行消峰调蓄计算,得到从什么时候到什么时候开始调蓄,控泄流量多少。前期当入库流量小于该流量时来多少泄多少(遵守过流能力约束),大于则进行消峰调蓄:超出控泄流量外的流量入库调蓄。当初始水位大于汛限水位时,前期泄洪降至汛限水位,以腾出调蓄库容;当初始水位小于汛限水位时,前期来多少泄多少,保持最高的调蓄库容。  

# 2)入库洪水  

上游水库直接采用相应流域产汇流模型结果、区间水库采用上游水库泄洪后马斯京根演进 $^+$ 区间流域入流得到。  

# 3)泄流计算  

基于水库库容曲线、溢洪道和泄洪洞水位\~泄流曲线计算水库调蓄过程,没有水位\~泄流曲线的根据堰流公式(溢洪道)和涵洞公式计  

算泄洪流量(泄洪洞);  

# (3)反向推演算法  

# 1)总体算法  

根据调度目标,按比例压缩确定共渠盐土庄闸(最大 $1600\mathrm{m}^{3}/5)$ 、汤永河四伏厂闸(最大 $300\mathrm{m}^{3}/5)$ 、安阳河郭盆闸控泄流量(最大$600\mathrm{m}^{3}/s)$ 的控泄流量,当在此控泄流量下,上游蓄滞洪区全部启用仍不满足(超过设计滞洪水位)时,放宽至最大控泄(这里省略了放宽至约束和最大控泄之间的值),试算依次启用小滩坡、任固坡蓄滞洪区。  

# 2)各蓄滞洪区优先级排序  

第 1 优先级:5 片打包和崔家桥、广润坡同级,5 片里按顺序:良相坡、共渠西、白寺坡、长虹渠、柳围坡。  

第 2 优先级:小滩坡  

第 3 优先级:任固坡  

3)闸站调度节点闸站按控泄流量调度,各蓄滞洪区进洪按规则调度。  

4)迭代试算顺序。  

迭代目标:除了满足调度目标外,还有各蓄滞洪区不超设计滞洪水位、河道不超最大过流能力。  

第 1 次:良相坡、共渠西下片、广润坡、崔家桥自然启用 –所有分洪闸、堰关闭;  

第 2 次:共渠西上片启用 --刘庄分洪堰开启  

第 3 次:白寺坡启用 –白寺坡分洪闸堰开启  

第 4 次:长虹渠启用 –小河口闸控泄最大到 $700\mathrm{m}^{3}/\mathsf{s}$ 、长虹渠分洪闸堰开启。(模拟方案先采用淇河卫河流量大,卫河流量下的洪水)  

第 5 次:柳围坡启用  –西沿村分洪堰、宋村分洪堰开启。  
第 6 次:调整控泄流量,小滩坡启用  –圈里分洪堰开启。  
第 7 次:调整控泄流量,任固坡启用  –北五陵分洪堰开启。  

# 3、输出结果  

1、调度目标是否能达到的结论描述,若不能达到给出原因;2、调度目标洪峰流量、流量过程、洪峰出现时间;3、各蓄滞洪区启用情况、总启用数量、最高滞蓄水位和滞洪量;4、各水库最高水位、泄洪流量过程、最大泄洪流量;5、分主次顺序的各闸站调度方案 (简短描述调度指令)和调度过程。  

# 4、其他说明  

1、除调度目标约束外,其他均为硬约束,在满足客观规律下一定遵守,不存在退而求其次的情况;
