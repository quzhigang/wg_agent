# 1 功能概况  

# 1.1编写目的  

此用户操作手册旨在向省水利厅、流域管理单位、市县水利局等水行政主管部门等相关系统使用人员介绍系统业务功能与操作流程,以便迅速、便捷掌握并使用系统,顺利开展各项业务。  

# 1.2文档范围  

测试方案预期给用户、测试人员、开发人员、项目管理者、其他质量管理人员和需要阅读本报告的高层经理阅读,为项目测试提供指导。

# 3 模型平台  

模型管理平台基本功能包括模型的分类编码、模型信息的全局预览、模型原理的查看、模型参数的修改、模型方案的增删改查、模型方案的计算暂停和停止控制、模型方案结果的查询、模型服务接口的管理等。  

# 3.1模型总览  

水利专业模型服务默认展示门户首页,实现对基础模型、模型实例、业务模型、模型方案,以及模型服务执行、调用的统计展示。  

(1)模型统计  

页面上方看板统计展示系统平台提供的基础模型、模型实例、业务模型、模型方案数量。  

(2)统计清单  

页面左侧看板统计展示系统内待计算、计算中、计算成功、计算失败等状态下的模拟方案数量。  

页面左侧看板统计展示系统提供的服务接口数量、服务接口呗调用的次数。  

(3)模型分布  

页面中间以地图、列表形式展示卫共流域模型建设情况。点击地图中流域区域,以弹窗形式展示该流域模型数量。点击放大或缩小实现地图的缩放,点击【复位】按钮实现地图恢复原始比例、位置。点击图例可选择显示与隐藏,显示图层可滑动调整透明度。  

(4)最新动态  

页面右侧看板以时间轴形式展示模型服务新增、修改、删除、执行、调用等操作。  

![](images/83cc79b04bbb664fb359891a71a46ece3e05d557f47943883d2246532943e68e.jpg)

页面下方以树状图形式将系统模型库按照模型分类分级展示。

# 5.2告警  

告警功能模块用于对流域范围内气象信息、实时雨情、水库和河道水情、闸站工情、工程安全、视频监测、遥感监测、AI监测等实时信息的展示查询,当实时监测数据超过安全阈值时系统自动发出告警,并将告警信息一键推送至相关防汛管理人员手机,辅助防洪决策。  

告警包括气象信息、雨情监测、水情监测、闸站监测、工情监测、生态监测、遥感监测、视频监测、AI监测、无人机等几大模块。

# 5.3预报  

预报是基础,根据卫共流域防洪业务需求,遵循客观规律,在总结分析“7.20暴雨”等典型历史事件和及时掌握现状的基础上,采用基于机理揭示和规律把握、产汇流 $+$ 河道一维水动力学耦合的洪水预报模型,新安江模型等水文模型,对水位、流量、水量、淹没影响、位移形变等水安全要素发展趋势做出不同预见期(短期、中期、长期等)的定量或定性分析,提高预报精度、延长预见期。

# 5.4预判  

预判功能基于水库、河道、蓄滞洪区的雨水情态势进行纳雨能力分析,根据短中期降雨预报和水情预报,预判洪水来源和组成情况,分析重要河段的潜在防洪风险以及工程实时防洪能力,通过参数调配评估当前水库、河道、蓄滞洪区未来可容纳降雨量,提供预报及联合会商研判的数据层面支撑。

# 1.新建方案

# 5.8.4 灾情评估  

灾情评估功能模块通过遥感影像分析、洪水位与地形数据叠加分析等手段,对实际洪水淹没情况进行分析,包括淹没面积、平均水深、最大水深、总滞洪量等,通过将社会经济数据展布到地图上,与洪水淹没区域进行空间叠加分析,实现洪灾损失评估,从而辅助洪水调度决策。
