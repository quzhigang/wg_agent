# 执行 (Execute): 根据计划进行实施

## 待执行的计划 (Plan to Execute)

读取计划文件：`$ARGUMENTS`

## 执行指令 (Execution Instructions)

### 1. 阅读并理解 (Read and Understand)

- 仔细阅读整个计划
- 理解所有任务及其依赖关系
- 记录需要运行的验证命令
- 审查测试策略

### 2. 按顺序执行任务 (Execute Tasks in Order)

对于 "分阶段任务" 中的每个任务：

#### a. 确定任务内容

- 确定所需的文件和操作
- 如果涉及修改，请阅读现有的相关文件

#### b. 实施任务

- 严格遵循详细的规格说明
- 与现有的代码模式保持一致
- 包含适当的类型提示 (Type hints) 和文档
- 在合适的地方添加结构化日志

#### c. 实时验证

- 每次修改文件后，检查语法
- 确保导入路径正确
- 验证类型定义是否妥当

### 3. 实施测试策略 (Implement Testing Strategy)

完成实施任务后：

- 创建计划中指定的所有测试文件
- 实施提到的所有测试用例
- 遵循概述的测试方法
- 确保测试涵盖边界情况

### 4. 运行验证命令 (Run Validation Commands)

按顺序执行计划中的 **所有** 验证命令：

```bash
# 完全按照计划中的说明运行每个命令
```

如果任一命令失败：

- 修复问题
- 重新运行该命令
- 仅在测试通过后继续

### 5. 最终验证 (Final Verification)

在完成前：

- ✅ 计划中的所有任务均已完成
- ✅ 所有测试已创建并已通过
- ✅ 所有验证命令均已通过
- ✅ 代码符合项目规范
- ✅ 已根据需要添加/更新文档

## 备注 (Notes)

- 如果遇到计划中未提及的问题，请记录下来
- 如果需要偏离计划，请解释原因
- 如果测试失败，请修复实施代码直至通过
- 切勿跳过验证步骤
