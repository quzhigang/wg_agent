# 卫共流域防洪"四预"相关接口详细文档 - 获取专业模型方案及结果数据相关接口

## 概述

本文档描述了卫共数字孪生流域获取专业模型方案及结果数据相关的API接口，共计24个接口。接口的输入输出参数均有完整描述，输出一般都有返回的结果样例，全部接口均基于ASP.NET项目构建，所有接口通过Model_Ser.ashx.cs处理。每个接口都包含详细的输入参数说明和输出结果格式，其接口地址统一为：<http://172.16.16.253/wg_modelserver/hd_mike11server/Model_Ser.ashx>

## 接口分类概览

本文档共包含 **24个API接口**，主要用于获取专业模型方案及结果数据。

---

## 接口列表

### 1. get_models

**用途**: 获取已有所有模型方案信息
**请求方式**: GET
**输入参数**:

- request_type: "get_models"
- request_pars: 模型实例名称字符串，默认为 "wg_mike11"
**输出结果**:
- JSON格式的模型信息字典，格式:{"方案id1":{"属性1":值1, "属性2":值2, ..}，"方案id2":{"属性1":值1, "属性2":值2, ..},...}，
  各方案属性相同，为固定的12个属性:
    plan_name(方案名称,varchar)
    plan_desc(方案描述,varchar)
    business_code(业务模型编码,varchar)
    business_name(业务模型名称,varchar)
    start_time(开始时间,varchar)
    end_time(结束时间,varchar)
    state(方案状态,varchar)
    step_save_minutes(模型结果保存时间步长(m),int4)
    expect_seconds(预估计算时长(s),int4)
    start_simulate_time(计算开始时间,varchar)
    progress_info(进度信息,text)
    view_point(相机位置,varchar)

### 4. get_gisgc_polygon_result

**用途**: 获取方案某时刻河道水面GIS面要素结果，为geojson格式的带Z值的三维水面要素，用于在三维场景中绘制三维水面
**请求方式**: GET
**输入参数**:

- request_type: "get_gisgc_polygon_result"
- request_pars: JSON数组格式: [plan_code, now_time]
  - 参数1: 方案ID (plan_code)
  - 参数2: 时间字符串 (now_time)
**输出结果**:
- JSON格式的GIS面结果，为geojson格式的带Z值的三维水面要素

### 5. get_sampleline_data_result

**用途**: 获取方案的GIS过程线的全过程属性结果，用于在地图区分色动态渲染过程结果，如流量、流速等
**请求方式**: GET
**输入参数**:

- request_type: "get_sampleline_data_result"
- request_pars: JSON数组格式: [plan_code, gis_restype]
  - 参数1: 方案ID (plan_code)
  - 参数2: 结果数据类型: "Waterlevel", "Speed", "Waterh", "Discharge"，即水位、流速、水深、流量
**输出结果**:
- JSON格式的GIS线数据属性结果，样式：
  {
      "0":        //时刻0
      [           //各线段的属性值，与样板线要素一一对应，如果结果数据类型为"Waterlevel"，则属性表示水位
          1.14,
          1.26,
          ...
      ],
      "1":       //时刻1
      [          //各线段的属性值
          1.14,
          1.26,
          ...
      ],
      ...
  }

### 6. get_gistj_result

**用途**: 获取方案的GIS统计线结果，为geojson格式的河道分段线要素，用于在地图区分色渲染全过程最大流量、流速等分布结果
**请求方式**: GET
**输入参数**:

- request_type: "get_gistj_result"
- request_pars: 方案ID字符串
**输出结果**:
- JSON格式的GIS统计线结果，为geojson格式的河道分段线要素

### 7. get_gistj_polygon_result

**用途**: 获取方案的GIS统计面结果(淹没面)，为geojson格式的二维面要素，用于在地图中分水渲染淹没区水深分布
**请求方式**: GET
**输入参数**:

- request_type: "get_gistj_polygon_result"
- request_pars: 方案ID字符串
**输出结果**:
- JSON格式的GIS统计面结果，为geojson格式的二维面要素

### 8. get_point_result

**用途**: 查询方案河道上某点的水位流量等结果(时间为""，则返回序列)，用于在地图中点击查询某位置结果信息
**请求方式**: GET
**输入参数**:

- request_type: "get_point_result"
- request_pars: JSON数组格式: [plan_code, now_time, jd, wd]
  - 参数1: 方案ID (plan_code)
  - 参数2: 时间字符串 (now_time) - 如果时间为""，则返回时间序列
  - 参数3: 经度 (jd)
  - 参数4: 纬度 (wd)
**输出结果**:
- JSON格式的点结果数据，包括该水位、水深、流速、流量
